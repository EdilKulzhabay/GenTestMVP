Реализована эволюция к Knowledge-component-centric архитектуре (бэкенд, аддитивно, ничего из старого не удалено). Все 4 фазы прошли `cd server && npx tsc --noEmit`.

**Фаза 0 (фиксы):** убран over-reuse `findLatestTestBySubject` → точный кэш по контенту (`aiService.computeTestContentHash`); кэш теста включён всегда (был отключён при OPENAI_API_KEY); версия промпта в ключе кэша урока (`LESSON_PIPELINE_VERSION`); summary урока персистится в `NodeLessonContent` (live-КТП) через `persistNodeLessonSummary` (не регенерится на каждый просмотр); анти-галлюцинационный `groundingRule` в промпте теста; в `recordTestSubmitted` освоенный узел больше не копит fail-счётчик.

**Фаза 1 — KnowledgeComponent (подтема):** встроен в `KtpTopic.knowledgeComponents[]` (стабильный `_id`, status proposed|confirmed). AI предлагает (`roadmapAIService.proposeKnowledgeComponents`), куратор подтверждает. Сервис `knowledgeComponent.service.ts`. Эндпоинты под `/ktp/:subjectId/topics/:topicId/components*`. Консолидация урока стала KC-aware (секция = KC, `lessons[].knowledgeComponentIds`).

**Фаза 2 — банк вопросов:** новая коллекция `QuestionItem` (`question_items`), тег `knowledgeNodeId`(=ktpTopicId)+`knowledgeComponentIds`+difficulty+sourceRefs+contentHash, unique-индекс дедупа. `questionBank.service.ts`: `coverage`, `generateForCoverage` (генерация под пробел + LLM-верификация `aiService.verifyQuestionItem` + дедуп), `assembleNodeTest` (выборка по покрытию KC → создаёт обычный `Test`, переиспользуя submit/grade). Admin: `/ktp/.../bank/coverage|generate`. Learner: `POST /tests/node-bank`.

**Фаза 3 — пер-KC mastery + адаптив:** коллекция `UserKcMastery` (`user_kc_mastery`). `Test` question получил `questionItemId`+`knowledgeComponentIds`. `userKcMastery.service.recordFromSubmission` вызывается в `submitTest`/`claimGuestTest` (no-op для обычных тестов без KC-тегов). Адаптивная сборка: слабые KC первыми, сложность под уровень, SR-деприоритет недавних item'ов (`recentItemIds`), ретайр сломанных вопросов по qualityStats.

**Верификация (2026-06-22):** сквозной smoke (`src/scripts/bankSmoke.ts <subjectId> <ktpTopicId>`) прошёл на реальной БД (Биология, KZ): propose→confirm→coverage→generateForCoverage→assembleNodeTest(10 вопросов, все с KC+itemId)→submit→пер-KC mastery (7/10 mastered). По пути починены 2 бага: (a) валидация батча KC-вопросов была all-or-nothing → `parseRegularQuestionsLenient` (валидные оставляем, битые дропаем); (b) LLM-судья `verifyQuestionItem` резал ~80% (субъективный критерий «нет второго верного») → сужен до «ключ верен + заземлён», borderline→ok, причины логируются. Запускать на subjectId с КТП+маппингом (Биология/История есть; Математика/языки — КТП пустой).

**Реализовано далее (ветка feat/knowledge-bank, коммиты 5deafc8 Part A, f07fdbb Part B):** (A) Админ-UI банка: новый read-эндпоинт `GET /ktp/:subjectId/topics/:topicId/bank/items` (isAdmin, отдаёт вопросы С правильными ответами через `questionBankService.listItems`); клиентские `ktpApi` методы KC/банка + типы; компонент `client/src/components/ktp/TopicKnowledgeBank.tsx` врезан per-topic в `KtpCatalogPage` (propose/confirm/delete KC, виджет покрытия, генерация банка, просмотр вопросов; мутации гейтятся `role==='admin'`). Правка/ретайр/удаление item'ов — оставлены TODO-семами. (B) Размер теста перестал быть «ровно 10»: `Test.model` валидатор `===10`→`1..120`; `assembleNodeTest` принимает `size` (5/10/15/20), size подмешан в реюз-хэш; `/tests/node-bank` принимает size; `/generate(+guest)` валидируют questionCount 1..120. Качество: `generateForCoverage` переписан на раунды-ретраи + разброс difficulty; `contentHashOf` теперь хеширует текст+варианты+correctOption; промпт дистракторов; BANK_PROMPT_VERSION→bank-v2. ⚠️ снятие `===10` также «оживляет» батч-ENT 20/40 (пробник) — проверить на регресс.

**Cutover ученика на банк + починка зачёта (сделано, ветки: server `feat/knowledge-bank`+main, edu-pwa `feat/edu-app-decomposed-be`):** Реальный фронт ученика = `edu-pwa/packages/edu-app` (Render, render.yaml), НЕ legacy `GenTestMVP/client`. Поток узла: Roadmap→Lesson→`RoadmapTestPage`→`useAuthorizedTestSession`→`useAuthorizedTestAcquisition`. **Открытие:** зачёт узла в edu-app был ДРЕМЛЮЩИМ — `roadmapSessionId` нигде не заполнялся, а `submitTest` начисляет (`recordTestSubmitted`→`UserRoadmapProgress.mastered`, единственный источник для `getPersonalRoadmap`) только если в `/tests/submit` оба `roadmapNodeId`+`roadmapSessionId` непустые. Mastery двигал только пробник. Починка: в `useAuthorizedTestAcquisition` чеканим `roadmapSessionId` (`roadmap-${nodeId}-${Date.now()}-${rand}`, свежий на попытку) при свежей сборке; `submitTestAnswers` инвалидирует `Roadmap`. Cutover: acquisition теперь **банк-первым** (`/tests/node-bank`, ktpTopicId = `parseKtpNodeId(nodeId)` снимает префикс `ktp:`) → при 400/403 LLM-фолбэк (`/tests/generate`, которому добавлен `roadmapNodeId` для guard). Бэк: `assembleNodeTest` получил `allowRefill` (default true); `generateNodeTestFromBank` зовёт с `allowRefill:false` — студенческий hot-path не делает синхронную LLM-догенерацию банка, быстрый 400→фолбэк. ⚠️ Включился 3-fail блок узла (`assertKnowledgeMapTestAllowed`); при блоке (403) тест-экран сейчас показывает дженерик «ошибка/повторить» (известный UX-гэп, снять блок — «Освоил» на уроке). Банк отдаёт только single_choice → фолбэк даёт смешанные типы.

**Расширение типов вопросов банка (сделано, bank-v3; ветки server `feat/knowledge-bank`+main, edu-pwa admin `feat/edu-app-decomposed-be`):** Банк больше не только single_choice. Микс: `BANK_TYPE_CYCLE` (per 10: single×5, multiple×2, short×1, matching_single×1, matching_multiple×1; **text_input исключён** — нечёткий грейдинг по ключевым словам → нечестный зачёт). `ai.generateKcQuestions` принял `typeMix` (смешанный промпт + пер-типовые правила `KC_TYPE_RULES`), парсит `parseEntQuestionsLenient` (best-effort по всем типам). `verifyQuestionItem` стал type-aware (`formatExpectedAnswer` + контекст matching). `contentHashOf` type-aware (single_choice хеш не изменился → без churn). `BANK_PROMPT_VERSION`→`bank-v3`. **Латентный баг починен:** `gradeAnswer` для `matching_single` ждал `{L:"R"}`, а edu-app всегда кодирует `{L:["R"]}` (`encodeMatchingAnswer`) → matching_single грейдился всегда неверно (в т.ч. в пробнике); теперь принимает обе формы. edu-admin `KtpTopicBank` рендерит ответы всех типов (`BankItemAnswer`, `BankItem.question` расширен). edu-app НЕ трогали — `mapGeneratedQuestion`/answer-компоненты/`answerCodec` уже рендерят+кодируют все типы (контракт доказан пробником `entSpec`). Деплой: **server + edu-admin** (edu-app не нужен). Существующие банки (все single) не затронуты; новые типы появляются при генерации куратором.

**НЕ подключено / следующее:** (1) задеплоить server (bank-v3 + matching_single fix) + edu-admin (Render); (2) 403 блок-экран узла — адресное сообщение вместо дженерик-ошибки (выбран как след. шаг, но переключились на типы); (3) полный CRUD вопросов банка (правка/ретайр/удаление) — только TODO-семы в коде, UI-кнопки задизейблены; (4) `Test` не переанкорен с book на node (`bookId` остаётся обязательным, ассембли берёт primary book узла); (5) фронт ученика size-выбор для node-bank не шлёт (бэкенд готов).
