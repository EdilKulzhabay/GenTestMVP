# Edu AI — Документация

## Обзор проекта

**Edu AI** — платформа для AI-генерации тестов по учебному контенту с автоматическим разбором ошибок и рекомендациями.

Преподаватель (администратор) загружает учебный контент через админ-панель или импортирует JSON-файл. Ученик выбирает предмет, книгу и главу, запускает генерацию теста. AI формирует вопросы по материалу, проверяет ответы, даёт развёрнутый анализ ошибок и ссылки на конкретные места в учебнике.

## Стек технологий

| Слой | Технологии |

## Роли пользователей

### Гость (без авторизации)
- Может пройти пробный тест
- Видит тизер результата (общий балл + краткий обзор)
- После регистрации/входа — тест автоматически сохраняется в историю

### Ученик (user)
- Полный доступ к тестам с AI-разбором ошибок
- История всех пройденных тестов
- Статистика (средний балл, лучший/худший результат)

### Администратор (admin)
- Управление учебным контентом (предметы, книги, главы, темы, параграфы)
- Импорт предметов из JSON-файла
- Создание контента вручную через формы

## Структура данных

### Subject (Предмет)
```
Subject
 ├── title — название предмета
 ├── description — описание
 └── books[] — массив учебников
      ├── title — название книги
      ├── author — автор
      └── chapters[] — массив глав
           ├── title — название главы
           ├── order — порядковый номер
           └── topics[] — массив тем
                ├── title — название темы
                └── paragraphs[] — массив параграфов
                     ├── order — порядковый номер
                     └── content
                          ├── text — текст параграфа
                          ├── pages[] — номера страниц в учебнике
                          └── metadata
                               ├── keywords[] — ключевые слова
                               ├── difficulty — сложность (easy/medium/hard)
                               └── source — источник
```

### User (Пользователь)
```
User
 ├── fullName — полное имя
 ├── userName — логин (уникальный)
 ├── email — почта (уникальная)
 ├── password — хеш пароля (bcrypt)
 ├── role — роль (admin / user)
 └── testHistory[] — история тестов
      ├── subjectId — ссылка на предмет
      ├── bookId — ID книги
      ├── chapterId — ID главы
      ├── answers[] — ответы пользователя
      │    ├── question — текст вопроса
      │    ├── selectedOption — выбранный вариант
      │    └── isCorrect — правильно ли
      ├── result
      │    ├── totalQuestions — всего вопросов
      │    ├── correctAnswers — правильных ответов
      │    └── scorePercent — процент
      ├── aiFeedback
      │    ├── summary — общий обзор от AI
      │    └── mistakes[] — разбор ошибок
      │         ├── question — вопрос
      │         ├── explanation — объяснение
      │         └── whereToRead — где перечитать
      │              ├── bookTitle
      │              ├── chapterTitle
      │              └── pages[]
      └── generatedQuestionsHash[] — хеши вопросов (для предотвращения повторов)
```

### Test (Сгенерированный тест — кэш)
```
Test
 ├── subjectId, bookId, chapterId
 ├── sourceContentHash — хеш контента (SHA-256, для кэширования)
 └── questions[]
      ├── questionText — текст вопроса
      ├── options[] — 4 варианта ответа
      ├── correctOption — правильный вариант
      ├── aiExplanation — объяснение от AI
      └── relatedContent — связанный контент
           ├── chapterId
           ├── topicId
           └── pages[]
```

## API Reference

**Base URL:** `/api/v1`

**Аутентификация:** JWT-токен в httpOnly cookie (устанавливается автоматически при login/register)

---

### Auth — Аутентификация

#### `POST /auth/register`
Шаг 1 регистрации — отправка 6-значного кода на email.

**Доступ:** Публичный

**Тело запроса:**
```json
{
  "fullName": "Иван Иванов",
  "email": "ivan@example.com",
  "userName": "ivan_i",
  "password": "secret123"
}
```

**Ответ (200):**
```json
{
  "success": true,
  "message": "Verification code sent to email"
}
```

**Ошибки:**
- `400` — Username или email уже заняты

---

#### `POST /auth/verify-email`
Шаг 2 — подтверждение кода и создание аккаунта.

**Доступ:** Публичный

**Тело запроса:**
```json
{
  "email": "ivan@example.com",
  "code": "123456"
}
```

**Ответ (201):**
```json
{
  "success": true,
  "message": "User registered successfully",
  "data": {
    "token": "eyJhbGciOiJI...",
    "user": {
      "id": "6789...",
      "fullName": "Иван Иванов",
      "userName": "ivan_i",
      "role": "user"
    }
  }
}
```

**Ошибки:**
- `400` — Неверный или просроченный код (15 минут)

---

#### `POST /auth/create-admin`
Создание администратора без email-верификации.

**Доступ:** Публичный (рекомендуется использовать только при первоначальной настройке)

**Тело запроса:**
```json
{
  "fullName": "Admin",
  "userName": "admin",
  "password": "adminpass"
}
```

**Ответ (201):** Аналогичен verify-email, `role: "admin"`

---

#### `POST /auth/login`
Вход в систему.

**Доступ:** Публичный

**Тело запроса:**
```json
{
  "userName": "ivan_i",
  "password": "secret123"
}
```

**Ответ (200):**
```json
{
  "success": true,
  "message": "Login successful",

    "token": "eyJhbGciOiJI...",

      "id": "6789...",
```

**Ошибки:**

## Пользовательские потоки

### Поток гостя
1. Открывает `/welcome` — экран приветствия
2. Нажимает «Попробовать как гость»
3. Выбирает предмет → книгу → главу
4. Запускается генерация теста (экран загрузки с состояниями)
5. Проходит тест (4 вопроса с вариантами ответов)
6. Видит тизер результата: общий балл + обрезанный summary
7. Нажимает «Войти» или «Зарегистрироваться»
8. После авторизации — тест автоматически сохраняется в историю
9. Перенаправляется на полный результат с AI-разбором

### Поток ученика
1. Входит в систему → дашборд с историей тестов
2. Выбирает предмет → книгу → главу (или «вся книга»)
3. Запускается генерация (AI создаёт вопросы по материалу)
4. Проходит тест, ответы автосохраняются
5. Получает полный результат:
   - Общий балл (% правильных)
   - AI-разбор каждой ошибки
   - Рекомендации: что перечитать, на каких страницах
6. Тест сохраняется в историю

### Поток администратора
1. Входит → дашборд со статистикой (количество предметов, книг, глав)
2. **Импорт JSON** — загружает готовый файл с полной структурой предмета
3. Или создаёт вручную: предмет → книга → глава → тема → параграф
4. Контент становится доступен для генерации тестов

## Развёртывание

### Переменные окружения (server/.env)

| Переменная | Обязательна | Описание |
|------------|:-----------:|----------|
| `MONGODB_URI` | Да | Строка подключения к MongoDB |
| `JWT_SECRET` | Да | Секрет для подписи JWT-токенов |
| `PORT` | Нет | Порт API (по умолчанию 5000) |
| `NODE_ENV` | Нет | Окружение (development / production) |
| `CORS_ORIGIN` | Нет | Разрешённые origins (`*` — все) |
| `OPENAI_API_KEY` | Нет | Ключ OpenAI (без него — mock-тесты) |
| `SMTP_HOST` | Нет | SMTP хост для email-верификации |
| `SMTP_PORT` | Нет | SMTP порт (465 для SSL, 587 для STARTTLS) |
| `SMTP_SECURE` | Нет | `true` для порта 465 |
| `SMTP_USER` | Нет | SMTP логин |
| `SMTP_PASS` | Нет | SMTP пароль |
| `SMTP_FROM` | Нет | Адрес отправителя |

### Переменные окружения (client/.env)

| Переменная | Описание |
|------------|----------|
| `VITE_API_URL` | URL бэкенда (например `/api/v1` для production через nginx) |

### Запуск

```bash
# Сервер
cd server
npm install
npm run build
pm2 start dist/server.js --name server

# Клиент
cd client
npm install
npm run build
# Скопировать dist/ в /var/www/client (для nginx)
```

### Первоначальная настройка

1. Запустить MongoDB и сервер
2. Создать админа: `POST /api/v1/auth/create-admin`
3. Войти как admin
4. Импортировать `subject.json` через кнопку «Импорт JSON» на дашборде
5. Ученики могут начинать тесты

## Формат ошибок

Все ошибки возвращаются в едином формате:

```json
{
  "success": false,
  "message": "Описание ошибки"
}
```

| Код | Описание |
|-----|----------|
| 400 | Bad Request — невалидные данные, дубликат |
| 401 | Unauthorized — не авторизован или невалидный токен |
| 403 | Forbidden — нет прав (не admin) |
| 404 | Not Found — ресурс не найден |
| 500 | Internal Server Error — ошибка на сервере |
